# API Documentation

Base path:

```txt
/api/:API_VERSION
```

For local development with `API_VERSION=v1`:

```txt
http://localhost:4001/api/v1
```

## Response Shape

Successful responses use:

```json
{
  "success": true,
  "message": "Message text",
  "data": {}
}
```

Validation errors use:

```json
{
  "success": false,
  "message": "Validation failed",
  "errors": {}
}
```

Operational errors use:

```json
{
  "success": false,
  "message": "Error message"
}
```

## Health

### GET `/health`

Checks whether the server is running.

Full local URL:

```txt
http://localhost:4001/health
```

Response:

```json
{
  "status": true,
  "message": "Server is healthy"
}
```

## Orders

Order routes are mounted at:

```txt
/api/v1/orders
```

### POST `/api/v1/orders/create-order`

Creates a new order.

Rate limit:

```txt
10 requests per minute
```

Request body:

```json
{
  "customerName": "Suresh",
  "phoneNumber": "9876543210",
  "items": [
    {
      "productName": "Wireless Mouse",
      "quantity": 2,
      "price": 499
    }
  ],
  "paymentStatus": "PAID",
  "orderStatus": "PLACED"
}
```

Rules:

- `customerName`: string, 2 to 100 characters
- `phoneNumber`: Indian mobile number matching `^[6-9]\d{9}$`
- `items`: at least one item
- `items[].productName`: string, 2 to 150 characters
- `items[].quantity`: positive integer
- `items[].price`: positive number
- `paymentStatus`: optional, one of `PENDING`, `PAID`, `FAILED`
- `orderStatus`: optional, one of `PLACED`, `PROCESSING`, `READY_TO_SHIP`

Notes:

- `orderId` is generated by the server.
- `amount` is calculated by the server from item quantity and price.

### GET `/api/v1/orders/get-orders`

Fetches orders with pagination, filters, and sorting.

Rate limit:

```txt
30 requests per minute
```

Query params:

| Param | Required | Values |
| --- | --- | --- |
| `page` | No | Positive integer |
| `limit` | No | Positive integer, max `100` |
| `orderStatus` | No | `PLACED`, `PROCESSING`, `READY_TO_SHIP` |
| `paymentStatus` | No | `PENDING`, `PAID`, `FAILED` |
| `sortBy` | No | `createdAt`, `amount`, `customerName`, `orderStatus`, `paymentStatus` |
| `order` | No | `asc`, `desc` |

Example:

```txt
GET /api/v1/orders/get-orders?page=1&limit=100&sortBy=createdAt&order=desc
```

Response data:

```json
{
  "orders": [],
  "total": 0
}
```

### GET `/api/v1/orders/get-order/:orderId`

Fetches a single order by generated order ID.

Rate limit:

```txt
30 requests per minute
```

Example:

```txt
GET /api/v1/orders/get-order/ORD-20260720-1DCCA3A1
```

## Scheduler

Scheduler routes are mounted at:

```txt
/api/v1/scheduler
```

### POST `/api/v1/scheduler/process-orders`

Runs the scheduler manually.

Required header:

```txt
x-scheduler-secret: <SCHEDULER_SECRET>
```

This endpoint is protected because it changes order statuses.

Response data:

```json
{
  "startedAt": "2026-07-20T12:40:00.002Z",
  "endedAt": "2026-07-20T12:40:00.107Z",
  "durationInMs": 105,
  "totalOrdersChecked": 4,
  "totalOrdersUpdated": 4,
  "failedOrders": 0,
  "status": "SUCCESS"
}
```

### GET `/api/v1/scheduler/logs`

Fetches scheduler execution logs.

This endpoint does not require `x-scheduler-secret` because it only reads logs.

Query params:

| Param | Required | Values |
| --- | --- | --- |
| `page` | No | Positive integer |
| `limit` | No | Positive integer, max `100` |
| `status` | No | `SUCCESS`, `FAILED`, `PARTIAL_SUCCESS` |
| `sortBy` | No | `createdAt`, `startedAt`, `endedAt`, `durationInMs`, `totalOrdersChecked`, `totalOrdersUpdated`, `failedOrders`, `status` |
| `order` | No | `asc`, `desc` |

Example:

```txt
GET /api/v1/scheduler/logs?page=1&limit=20&sortBy=createdAt&order=desc
```

Response data:

```json
{
  "logs": [],
  "total": 0
}
```
